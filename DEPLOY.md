# Руководство по развертыванию (Deployment Guide)

Этот проект использует Next.js App Router, PostgreSQL (Prisma), JWT-авторизацию и поддерживает загрузку изображений как локально, так и через Cloudinary.

## Локальный запуск (Development)

1. **Создайте файл `.env` в корне проекта.**
   Скопируйте содержимое `.env.example`:
   ```bash
   cp .env.example .env
   ```

2. **Настройте переменные окружения:**
   ```env
   # PostgreSQL Connection String (укажите свои данные)
   DATABASE_URL="postgresql://user:password@localhost:5432/barakat_db?schema=public"

   # Секретный ключ для JWT сессий (сгенерируйте случайную строку)
   JWT_SECRET="ваша_супер_секретная_строка_минимум_32_символа"

   # (Опционально) Cloudinary для хранения изображений
   # CLOUDINARY_URL="cloudinary://my_key:my_secret@my_cloud_name"
   ```

3. **Установите зависимости:**
   ```bash
   npm install
   ```

4. **Примените миграции базы данных:**
   Эта команда создаст таблицы в вашей базе:
   ```bash
   npm run prisma:migrate
   ```

5. **Заполните базу начальными данными (Seed):**
   *(Внимание: Эта команда читает старые JSON файлы в папке `data/` и переносит их в PostgreSQL. Пароли хешируются с помощью bcrypt.)*
   ```bash
   npm run db:seed
   ```

6. **Запустите проект:**
   ```bash
   npm run dev
   ```
   Откройте `http://localhost:3001` в вашем браузере.

## Запуск в Production (VPS / Server)

В production-среде необходимо предпринять дополнительные шаги безопасности.

### 1. Подготовка окружения
На вашем сервере создайте `.env` файл с *реальными* секретами. Никогда не коммитьте `.env` в репозиторий.
```env
DATABASE_URL="postgresql://prod_user:strong_password@localhost:5432/prod_db?schema=public"
JWT_SECRET="c6b8...[сгенерируйте_длинную_строку]...92a1"

# Если вы используете локальную загрузку картинок, убедитесь что папка `public/uploads` имеет права на запись.
# Если вы используете Cloudinary, раскомментируйте и укажите URL:
# CLOUDINARY_URL="cloudinary://..."
```

### 2. Загрузка и сборка проекта
```bash
npm install
npx prisma generate
npm run build
```

### 3. Применение миграций в Production
**Внимание:** В production нельзя использовать `prisma:migrate` (которая делает `migrate dev`), так как она может удалить данные. Вместо этого используйте безопасную команду:
```bash
npx prisma migrate deploy
```

> **О Seed в Production:** Обычно `npm run db:seed` не запускают на продакшене автоматически, чтобы не перетереть данные или не загрузить тестовый мусор. Если вам нужно перенести JSON один раз — выполните команду вручную, но потом удалите `data/` или больше не используйте seed-скрипт.

### 4. Запуск сервера
Рекомендуется использовать PM2 или Docker для поддержания работы процесса:
```bash
# Стандартный запуск
npm run start

# Запуск через PM2 (убедитесь, что PM2 установлен: npm i -g pm2)
pm2 start npm --name "barakat-app" -- run start
```

## Защита и Безопасность

Проект уже включает базовые меры безопасности:
1. **Хеширование паролей:** Используется `bcryptjs` (salt rounds: 10).
2. **Сессии (JWT):** Используются безопасные `httpOnly` cookie (`sameSite=lax`). Флаг `secure` включается автоматически, если `NODE_ENV === "production"`.
3. **Защита данных (RBAC):** Пользователи могут удалять/редактировать только свои собственные объявления. Сервер проверяет владельца перед каждым `PATCH` и `DELETE` запросом.
4. **Защита от брутфорса:** Встроен простейший in-memory Rate Limiter для `/api/auth/login` и `register`.
   *Для крупных проектов (multi-instance) In-Memory лимитер работать не будет (каждый инстанс Node.js имеет свою память). Рекомендуется перейти на Redis (например, пакет `@upstash/ratelimit`).*
5. **Валидация (Zod):** Все входящие данные для авторизации и объявлений строго валидируются через Zod, предотвращая SQL-инъекции и NoSQL-атаки типа отправки непредвиденных структур данных.
